# Kelsier Routing Audit

**Date:** 2026-06-18  
**Scope:** `claude/agents/synod-kelsier.md`, `claude/CLAUDE.md`, `claude/charter-details.md`  
**Question:** Is synod-kelsier needed, given that Sazed routes to agents directly in practice?

---

## Background

Sazed's routing posture was dialled to **advisory** as of commit `e859969` (2026-06-18): suggest a specialist, wait for user approval, auto-dispatch only for multi-discipline or high blast-radius cases. In practice, most tasks are single-discipline and Sazed handles them without Kelsier.

---

## Kelsier's Three Defined Roles

### 1. Multi-discipline routing

Kelsier holds a keyword→agent decision table (`synod-kelsier.md:46–58`) that maps request keywords to specialists. The charter triggers this path with: *"Route through synod-kelsier if ambiguous across 2+ disciplines."*

**Finding: largely redundant.** The same keyword→agent logic already lives in Sazed's CLAUDE.md routing rules. When Sazed invokes Kelsier for a multi-discipline task, Kelsier applies an identical table and returns a routing plan Sazed could have produced itself. This path is also bypassed in the vast majority of sessions because most tasks are single-discipline.

### 2. Conflict mediation

When two non-veto agents disagree, Kelsier mediates before the conflict reaches the user. When two veto-holders conflict (e.g. Elend vs. TenSoon), Kelsier collects both positions and presents them together as a unified escalation — preventing two independent veto escalations from landing separately (`charter-details.md:25–28`).

**Finding: genuine value, rarely invoked.** This function is distinct from anything Sazed does. The cost of keeping it is low; the cost of losing it is a messier conflict-resolution path if a veto-vs-veto situation ever fires.

### 3. Cascading halt notification

The charter states Kelsier "must be notified" when any specialist halts, so it can detect simultaneous conflicting vetoes and suspend declared dependents (`charter-details.md:35–36`).

**Finding: aspirational.** No automatic notification mechanism exists. This control requires explicit invocation by Sazed or the user. It functions only if the routing plan was generated by Kelsier in the first place (so dependency relationships are declared).

| Role | Status | Notes |
|------|--------|-------|
| Multi-discipline routing | Redundant | Same logic as Sazed's charter; silently skipped in most sessions |
| Conflict mediation | Genuine value | Distinct function; keeps veto-vs-veto escalations unified |
| Cascading halt notification | Aspirational | Requires explicit invocation; no automatic trigger |

---

## Options

### Option A — Keep Kelsier, tighten the routing trigger
Remove the "multi-discipline ambiguous" routing trigger from the charter. Kelsier is invoked **only** when (a) two or more specialists have been named and a conflict is anticipated, or (b) a veto has fired and cascading halt applies.

### Option B — Strip routing role, keep Kelsier as pure mediator *(recommended)*
Remove the keyword→agent table from `synod-kelsier.md`. Reframe Kelsier's mandate as: conflict mediation, veto-conflict synthesis, and cascading halt coordination. His definition becomes shorter and his purpose becomes legible. This makes the charter honest about what he actually does.

### Option C — Retire Kelsier entirely
Fold the mediation role into Sazed's escalation language. The existing escalation protocol (*"This requires your decision, Mistborn."*) can present both veto positions without a middleman. Appropriate only if veto-vs-veto conflicts are confirmed absent from real sessions.

---

## Recommendation

**Option B.** Kelsier's routing role is redundant with Sazed's charter and is already being silently skipped. Removing it makes the system honest. His mediation role is small but structurally sound — it gives conflict resolution a named, write-blocked handler that is distinct from Sazed's narrator posture. Retiring him entirely loses that clean separation at minimal benefit.

**Suggested edit:** update `claude/agents/synod-kelsier.md` to remove the keyword routing table and reframe the opening mandate, core function, and output format around mediation and cascading halt only. No changes required to CLAUDE.md or charter-details.md for Option B.
